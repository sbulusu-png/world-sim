## What is EUROSIM?

EUROSIM is a real-time geopolitical simulation where **six AI-controlled European nations** interact autonomously on an interactive SVG map. Each nation has a distinct personality, maintains trust scores toward every other nation, remembers past interactions, and uses a **Large Language Model (Meta LLaMA 3.1 8B via Featherless AI)** to make strategic decisions.

You can trigger events (attacks, trade deals, alliances, sanctions) and watch the AI nations react — or start the autonomous simulation and observe emergent geopolitical narratives unfold in real time.

### Key Features

- **Real AI Decisions** — Each nation's response is generated by LLaMA 3.1 8B with full context (trust, alliances, memory, personality)
- **Transparent AI** — Every decision shows its source (AI / Rule-based / Autonomous) with full reasoning visible in the Strategic Analysis panel
- **6 Nations with Distinct Personalities** — France (diplomatic), Germany (diplomatic), UK (opportunistic), Russia (aggressive), Poland (defensive), Italy (opportunistic)
- **Trust + Alliance + Memory Systems** — Three interlocking systems create emergent behavior: trust shifts influence alliances, alliances affect decisions, memories create grudges and loyalties
- **Real-World Event Injection** — Bright Data integration pulls real geopolitical news to influence the simulation
- **Military Command Center UI** — Dark-themed interface with animated event arrows, nation dossiers, and live debug overlay
- **Graceful AI Fallback** — If the AI API fails, a comprehensive rule-based system takes over seamlessly

## Getting Started

### Prerequisites

- **Node.js** 18+ (recommended: 20+)
- **npm** 9+
- A **Featherless AI** API key — get one at [featherless.ai](https://featherless.ai)
- *(Optional)* A **Bright Data** API key for real-world event injection

### 1. Clone the Repository

```bash
git clone https://github.com/your-username/world-sim.git
cd world-sim
```

### 2. Set Up Environment Variables

```bash
cp .env.example .env
```

Edit `.env` and add your API keys:

```env
# Required — powers AI nation decisions
FEATHERLESS_API_KEY=your_featherless_api_key_here

# Optional — enables real-world event injection
BRIGHTDATA_API_KEY=your_brightdata_api_key_here

# Server port (optional, defaults to 3001)
PORT=3001
```

### 3. Install Dependencies

```bash
# Install server dependencies
cd server && npm install && cd ..

# Install client dependencies
cd client && npm install && cd ..
```

### 4. Start the Application

**Terminal 1 — Start the server:**
```bash
node server/index.js
```
You should see:
```
🌍 World Sim server running on port 3001
🏗️  World initialized with 6 nations
```

**Terminal 2 — Start the client:**
```bash
cd client && npx vite --port 5173
```

Open **http://localhost:5173** in your browser.

## How to Use

### The Interface

The UI is divided into four main areas:

| Area | Location | Purpose |
|------|----------|---------|
| **Top Bar** | Top | Turn counter, simulation controls (Start/Pause/Reset), time display |
| **Command Panel** | Left | Select an action (Attack, Trade, Ally, Sanction, Support, Betray) and a target nation |
| **Map View** | Center | Interactive SVG map of Europe — click a nation to select it. Animated arrows show events flowing between nations |
| **Side Panel** | Right | **Nation Dossier** (trust bars, alliances, memory patterns) and **AI Strategic Analysis** (the AI's reasoning for each nation) |

### Triggering Events

1. **Click a nation** on the map (e.g., France) — it becomes your "actor"
2. **Select an action** from the Command Panel (e.g., "Attack")
3. **Select a target** nation (e.g., Russia)
4. **Click Execute** — the event is sent to the server

The AI will then generate reactions from all affected nations. Watch the map for animated arrows and check the Side Panel for AI reasoning.

### Available Actions

| Action | Trust Effect | Resource Cost | Description |
|--------|-------------|---------------|-------------|
| **Attack** | -30 | -15 | Military strike — major trust loss |
| **Trade** | +10 | +5 | Economic deal — slight trust gain, earns resources |
| **Ally** | +25 | 0 | Form/strengthen alliance — significant trust boost |
| **Sanction** | -15 | -5 | Economic pressure — moderate trust loss |
| **Support** | +20 | -5 | Diplomatic support — good trust gain |
| **Betray** | -40 | -10 | Break trust — devastating to relationships |

### Autonomous Simulation

Click **"Start Simulation"** in the Top Bar to let all nations act independently. The AI will:
- Evaluate each nation's situation every 7 seconds
- Generate autonomous decisions based on personality, trust, alliances, and memory
- Inject world events (from Bright Data or fallback pool) periodically
- Create emergent narratives — watch alliances form, wars break out, and betrayals unfold

Click **"Pause"** to stop, or **"Reset"** to return to the initial state.

### Debug Console

A collapsible debug overlay appears in the **bottom-left corner**. Click it to see:
- **API Status**: Green dot = AI connected, Red = using fallback only
- **AI Success Rate**: Percentage of decisions made by AI vs. fallback
- **Call Counts**: Total API calls, successes, and failures
- **Decision Breakdown**: Per-turn source tracking (AI / Rule / Self)

This proves the AI is genuinely making decisions — not faked.

## API Reference

| Method | Endpoint | Description |
|--------|----------|-------------|
| `GET` | `/api/state` | Current world state with enriched patterns + debug stats |
| `POST` | `/api/event` | Trigger an event: `{ actor, action, target, description? }` |
| `POST` | `/api/simulate/start` | Start autonomous simulation loop |
| `POST` | `/api/simulate/pause` | Pause simulation |
| `POST` | `/api/reset` | Reset world to initial state |

### Example: Trigger an Event

```bash
curl -X POST http://localhost:3001/api/event \
  -H "Content-Type: application/json" \
  -d '{"actor": "france", "action": "attack", "target": "russia"}'
```

### Example: Get World State

```bash
curl http://localhost:3001/api/state | jq '.nations[0].trust'
```

## How the AI Works

1. **Prompt Construction** — For each nation decision, the system builds a context-rich prompt including:
   - The nation's personality type and description
   - Current trust scores toward all other nations
   - Active alliances and their strength levels (★ to ★★★)
   - Memory of past interactions + pattern analysis (hostile/friendly counts)
   - The triggering event details
   - Any active world events from Bright Data

2. **AI Call** — The prompt is sent to **Meta LLaMA 3.1 8B Instruct** via the Featherless AI API with a 12-second timeout

3. **Response Parsing** — The AI's JSON response is validated for:
   - Valid action type (attack, trade, ally, sanction, support, betray, neutral)
   - Valid target nation ID
   - Reasoning string present

4. **Fallback** — If the AI call fails (timeout, invalid JSON, bad action), a rule-based system makes the decision using:
   - Trust thresholds (< -30 → hostile response)
   - Memory patterns (3+ hostile interactions → escalation)
   - Personality modifiers (aggressive nations have lower attack thresholds)
   - Resource awareness (low resources → prefer trade/neutral)

5. **State Update** — The decision updates trust scores, alliance states, memory logs, and resource counts. State validation runs after every mutation.

## Configuration

Key tuning parameters (in source):

| Parameter | File | Default | Description |
|-----------|------|---------|-------------|
| Simulation interval | `simulation.js` | 7000ms | Time between auto-simulation cycles |
| Max actions/cycle | `simulation.js` | 2 | Max nation reactions per cycle |
| AI timeout | `agent-loop.js` | 12000ms | Per-nation AI call timeout |
| Trust range | `state-validator.js` | [-100, 100] | Trust score bounds |
| Resource range | `state-validator.js` | [0, 120] | Resource bounds |
| Max memory | `state-validator.js` | 20 | Max memory entries per nation |
| Max event log | `state-validator.js` | 200 | Max global event log entries |
| Poll interval | `useWorldState.js` | 2000ms | Client polling frequency |
